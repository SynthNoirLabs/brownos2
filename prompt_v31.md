## 1. What We Tested This Round

We pursued a new structural hypothesis: what if `sys8` expects a "raw minted capability" directly from another syscall, without us unwrapping or modifying it via intermediate lambdas?

Previously, we used **adapters** to unwrap `Left(payload)` and pass the payload to `sys8`. For example:
`backdoor(nil)(λeither. either(sys8)(fail_handler))`

This time, we tested **`sys8` as the immediate CPS continuation**:
`(((syscall arg) 8) PSE)`

In this shape, the `syscall` evaluates to a minted result (e.g., `Left(3-way-list)` for `readdir`), and applies `sys8` to that raw result. Then `PSE` (our Either observer) catches whatever `sys8` returns.

## 2. Analysis & Synthesis

The "raw minted result term as capability" hypothesis is now definitively **DEAD**.

1. **No hidden pointer/provenance checks**: By passing the unmodified term minted directly by the VM, we ruled out the possibility that the VM tracks internal memory addresses or tags on the terms. The VM treats a forged `Left(pair)` exactly the same as the real `Left(pair)` generated by `sysC9`.
2. **`readdir` 3-way list does not bypass the gate directly**: The strongest candidate for a unique type shape was the 3-way Scott list from `readdir`. Passing it directly to `sys8` still yields `Right(6)`.
3. **The gate is strictly logical/functional**: We must conclude that `sys8` is operating strictly based on Lambda Calculus reduction semantics. It evaluates its argument until it matches a very specific structural condition, or expects its argument to evaluate *something else* (e.g. `sys8` passes a secret to the argument, which must return the correct response).

## 3. What is left? (Theories 2, 3, and 4 from v30)

We have exhausted direct data passes (both forged and raw). The remaining viable paths are the functional/structural theories from our previous Oracle session:

### A. The "Execution Context" (Theory 3)
Instead of passing the backdoor pair *to* `sys8`, we must use the backdoor pair to *wrap or execute* `sys8`.
- `backdoor_pair(sys8)(credential)`
- `A(sys8)` or `B(sys8)`
- If `sys8` is functionally an environment variable modifier, maybe it needs to be called from inside a specific scope.

### B. The "Login Function" ADT (Theory 2)
The argument isn't just a raw syscall output, but a manually constructed specific tree that satisfies a multi-part credential check.
- `cons("gizmore", cons("ilikephp", cons(access_log_bytes, nil)))`
- The challenge is building this without hitting the "Term too big!" payload limit.

### C. "Index Laundering" via Internal Lambdas (Theory 4)
The ultimate "dark magic" solution. The 2016 hinting around "crucial properties of the codes" and "don't be too literal with the ??s" might refer to dynamically generated de Bruijn indices.
- By Scott-destructing a value, lambdas are added to the evaluation context.
- A term with `Var(251)` passed into a destructing function gets shifted. If it hits `253` (FD), `254` (FE), or `255` (FF), the VM might interpret it as a built-in macro or capability marker that we couldn't type directly on the wire.
